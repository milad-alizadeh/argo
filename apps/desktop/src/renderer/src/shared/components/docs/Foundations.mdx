# Foundations

The three token families every cockpit surface is built from. Each specimen below is drawn
by the live component and the live utility classes and its list is read off the contract in
code — so this page cannot describe a system the app no longer has. If a role is added,
retired or retuned, the page changes with it.

Raw values live in `styles/argo-tokens.css`; `styles/globals.css` wires them into the
utilities shown here. Nothing on this page states a measurement of its own.

## Type ramp

`Text` is the one way type reaches the screen, and its variant map — `TYPE_ROLE_CLASS` — is
the only place a `text-<role>` class is ever spelled. The ladder is closed: a new rung needs
a job an existing one cannot do. Colour is deliberately not part of a role, which is why the
same sample reads identically in every row below.

<div className="flex flex-col gap-inset">
  {TYPE_ROLES.map((variant) => (
    <div className="flex items-baseline gap-inset" key={variant}>
      <Text variant="meta" className="w-28 shrink-0 text-foreground-faint">
        {variant}
      </Text>
      <Text variant={variant}>{SPECIMEN}</Text>
    </div>
  ))}
</div>

## Tone palette

Colour is owned by state, never by a component. `RAIL_TONES` is the closed set, and a tone's
name **is** its `--tone-*` token — a View interpolates `text-tone-${tone}` straight from the
derivation, so no component keeps a colour map of its own. A tone is never the only carrier
of meaning: it always arrives beside the word it tints.

<div className="flex flex-wrap items-start gap-region">
  {RAIL_TONES.map((tone) => (
    <span className="flex w-16 flex-col items-center gap-gap" key={tone}>
      <StatusDot tone={tone} />
      <Text variant="meta" className="text-foreground-faint">
        {tone}
      </Text>
    </span>
  ))}
</div>

## Spacing scale

Six named steps, reached as `gap-*`, `p-*`, `w-*` and `size-*`. Roles, not values: `p-inset`
survives a redesign that renumbers every pixel, `p-3` does not. Each bar below is drawn at
the width of its own token, so the ramp is the contract rather than a picture of it.

<div className="flex flex-col gap-gap">
  {SPACING_ROLES.map(([role, width]) => (
    <div className="flex items-center gap-inset" key={role}>
      <Text variant="meta" className="w-28 shrink-0 text-foreground-faint">
        {role}
      </Text>
      <span className={`h-gap rounded-sm bg-primary ${width}`} />
    </div>
  ))}
</div>
