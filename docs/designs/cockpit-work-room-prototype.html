Every billable action emits a usage event; the ledger is the single source of truth the invoice run reads from. Today events are counted ad-hoc per feature — this centralises them.

Acceptance criteria

  • Events are idempotent on (actor, action, idempotency_key).
  • The nightly invoice run reconciles against the ledger with zero drift on the fixture set.
