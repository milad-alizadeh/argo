# Cockpit ‚Äî component inventory

The build contract extracted from the settled study `cockpit.html` (per
`rules/design-studies.md` "Ship the inventory"). `/componentize-design` consumes this
file to scaffold the real components ‚Äî names, tiers, and props below are fixed; the
study's markup/CSS are disposable. Behavior ‚Äî ribbon rules R1‚ÄìR15, the session state
table S0‚ÄìS11, and pane anatomy ‚Äî lives in `cockpit-matrix.md`; this file covers only
*what* the components are.

Coverage: the study carries **31 distinct `data-component` regions**; every one appears
exactly once below. Recurring primitives the study renders without a `data-component`
attribute (the icon helper, dots, chips, buttons, tool/message/check rows, the finding
card, the screen itself) are included and marked **unnamed in study** ‚Äî `/componentize-design`
should treat their names here as proposals to freeze.

Tiering follows `rules/ui-components.md`: atoms ‚Üí molecules ‚Üí organisms ‚Üí screen,
grouped one table per tier.

## Atoms

All atoms are unnamed in study (the study renders them via helpers/classes, not
`data-component` regions).

**Text is under everything below.** Wherever a Composed-of column names a word, a label, a
count, a path, a timestamp or prose, that string is rendered by the Text atom ‚Äî the tables
name the copy, not a second way of setting type. The only exceptions are the two primitives
whose `asChild` hands their children to a Slot (Badge, Button); those compose the role class
from Text's map instead of wrapping.
