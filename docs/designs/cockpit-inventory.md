# Cockpit — component inventory

The build contract extracted from the settled study `cockpit.html` (per
`rules/design-studies.md` "Ship the inventory"). `/componentize-design` consumes this
file to scaffold the real components — names, tiers, and props below are fixed; the
study's markup/CSS are disposable. Behavior — ribbon rules R1–R15, the session state
table S0–S11, and pane anatomy — lives in `cockpit-matrix.md`; this file covers only
*what* the components are.

Coverage: the study carries **31 distinct `data-component` regions**; every one appears
exactly once below. Recurring primitives the study renders without a `data-component`
attribute (the icon helper, dots, chips, buttons, tool/message/check rows, the finding
card, the screen itself) are included and marked **unnamed in study** — `/componentize-design`
should treat their names here as proposals to freeze.

Tiering follows `rules/ui-components.md`: atoms → molecules → organisms → screen,
grouped one table per tier.

## Atoms

All atoms are unnamed in study (the study renders them via helpers/classes, not
`data-component` regions).
